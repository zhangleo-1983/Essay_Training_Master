# 联调与发布清单

## 1. 目标

用于安排正式环境联调和测试时的执行清单，按“上线前准备 -> 联调 -> 验收 -> 收尾”组织。

## 2. 上线前准备

### 代码与仓库

- [ ] 当前分支代码已推送远端
- [ ] 需要联调的提交号已固定
- [ ] README 与 docs 已同步最新状态

### 环境变量

- [ ] 前端 `VITE_API_BASE_URL` 已指向联调环境
- [ ] 后端 `ALLOWED_ORIGIN` 已配置
- [ ] 后端 `AUTH_MODE` 已确定
- [ ] 联调用 API key 已生成
- [ ] 模型 provider key 已配置
- [ ] `MODEL_NAME` 已确认

### 基础设施

- [ ] 前端访问地址可打开
- [ ] 后端服务已启动
- [ ] SQLite 数据目录可写
- [ ] `/api/health` 可访问

## 3. 联调执行

### 主链路

- [ ] 创建作文
- [ ] 发送消息
- [ ] 草稿自动保存
- [ ] 阶段切换
- [ ] 刷新恢复
- [ ] 打开历史会话
- [ ] 删除并恢复会话

### 治理能力

- [ ] 限流行为符合预期
- [ ] 模型调用日志可查
- [ ] 用量报表可查
- [ ] 预期错误提示清晰

## 4. 验收结论

### 必须通过

- [ ] 无阻塞性 P0 问题
- [ ] 无数据丢失问题
- [ ] 无模型密钥暴露问题
- [ ] 无明显跨域或鉴权错误

### 可延后

- [ ] 会话列表搜索体验
- [ ] 更完整的 smoke test 自动化
- [ ] 历史依赖清理

## 5. 联调后收尾

- [ ] 记录实际使用的 provider、model、域名和 key 策略
- [ ] 记录发现的问题及优先级
- [ ] 输出联调结论
- [ ] 确定下一阶段开发范围
