# 后续规划

## 1. 当前状态判断

项目已经从“前端原型”进入“后端基础设施已具备”的阶段，但还没有成为真正可试用的完整产品。最关键的短板不是后端能力，而是前后端主链路尚未正式打通。

## 2. 总体目标

按照“先闭环、再训练、再运营、最后规模化”的顺序推进。

## 3. 第一阶段：打通真实可用闭环

目标周期：1-2 周

核心任务：

- 前端切到后端 API
- 去掉前端直连模型逻辑
- 会话、草稿、阶段和消息全部走后端
- 增加基础错误提示和加载态
- 增加最小端到端联调测试

阶段结果：

- 用户能从输入题目开始，一路完成完整写作流程
- 前端不再暴露任何模型密钥

## 4. 第二阶段：把“聊天工具”升级成“训练产品”

目标周期：2-4 周

核心任务：

- 给五个写作阶段增加结构化产出
- 增加题库与题目分类
- 引入作文 rubric 反馈
- 增加历史记录页
- 增加学段与难度配置

阶段结果：

- 产品从“能聊”提升到“能训练”
- 用户可以看到过程沉淀与能力提升路径

## 5. 第三阶段：建立正式用户体系与管理能力

目标周期：3-6 周

核心任务：

- 用户注册登录
- 学生、家长、教师角色体系
- 布置题目与查看进度
- 管理员用量与成本面板
- 数据权限与操作审计

阶段结果：

- 项目具备真实学校/家庭试用的基础能力

## 6. 第四阶段：生产化与规模化

目标周期：1-2 个月

核心任务：

- SQLite 升级 PostgreSQL
- 限流升级 Redis
- 监控、告警、日志治理
- 成本控制与 provider 策略调度
- 内容安全与合规策略

阶段结果：

- 项目可进入正式生产部署评估

## 7. 我建议的优先级

### P0

- 前端接后端 API
- 去除前端模型直连
- 增加联调与回归验证

### P1

- 结构化训练成果物
- 历史记录
- rubric 反馈

### P2

- 正式登录体系
- 教师端 / 家长端
- PostgreSQL + Redis

## 8. 里程碑建议

- 里程碑 A：可闭环试用版
- 里程碑 B：可训练评估版
- 里程碑 C：可管理运营版
- 里程碑 D：可正式上线版
