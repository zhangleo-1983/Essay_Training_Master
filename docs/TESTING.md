# 测试说明

## 1. 测试目标

当前测试目标不是覆盖所有边界，而是确保学生单人版 MVP 的主链路稳定可用。

## 2. 当前已验证项

### 代码级验证

- `npm run lint`
- `npm run build`

### 后端能力验证

- 健康检查
- 会话创建、读取、列表、删除、恢复
- 草稿保存
- 阶段切换
- 模型调用日志
- 用量报表
- API key 鉴权
- SQLite 限流

### 前端主链路验证

- 前端已切到后端 API
- 新建作文会话
- 自动恢复最近会话
- 草稿自动保存
- 阶段切换
- 最近会话列表、删除、恢复

## 3. 正式环境联调测试范围

### P0 主链路

- 打开首页
- 创建作文会话
- 发送至少 3 轮消息
- 编辑草稿并自动保存
- 切换至少 2 个写作阶段
- 刷新页面后恢复当前会话
- 从会话列表重新打开历史作文
- 删除并恢复一篇作文

### P1 稳定性

- 后端未配置模型 key 时的报错提示
- 错误 provider 或错误模型名时的提示
- 限流触发时的前端提示
- 后端重启后的数据恢复
- 长草稿保存后的页面稳定性

### P2 运维联动

- `/api/health` 检查
- 数据库文件写入是否正常
- 模型调用日志是否持续产生
- 用量报表是否能查询到新数据

## 4. 建议测试方法

### 本地开发

- 前端手工 smoke test
- 后端接口手工 smoke test

### 正式联调环境

- 浏览器真实操作验证
- 管理员账号查看日志和报表
- 选定一个真实模型 provider 做完整跑通

## 5. 手工测试用例

### 用例 1：新建作文

前置条件：

- 前后端都已启动
- 模型 provider 可用

步骤：

1. 输入题目
2. 选择学段
3. 点击“开始新的写作训练”

期望结果：

- 成功进入写作页
- 出现初始教练消息
- 会话列表出现新作文

### 用例 2：发送消息

步骤：

1. 输入一条学生消息
2. 点击发送

期望结果：

- 显示加载态
- 返回教练回复
- 会话更新时间刷新

### 用例 3：自动保存草稿

步骤：

1. 在草稿区输入内容
2. 停止输入 1 秒左右

期望结果：

- 状态从“保存中”变成“已保存”
- 刷新页面后草稿仍存在

### 用例 4：阶段切换

步骤：

1. 点击其他写作阶段

期望结果：

- 当前阶段更新
- 刷新后阶段仍保持

### 用例 5：删除与恢复

步骤：

1. 删除当前会话
2. 在会话列表中恢复

期望结果：

- 删除后会话标记为已删除
- 恢复后可继续打开使用

## 6. 缺陷优先级建议

- P0：无法创建会话、无法发消息、无法保存草稿、刷新丢数据
- P1：删除恢复异常、阶段切换异常、报错提示不清晰
- P2：样式问题、非核心文案问题、少量性能抖动

## 7. 联调通过标准

- P0 用例全部通过
- 连续 10 次消息交互无致命错误
- 刷新恢复至少成功 3 次
- 删除/恢复流程至少成功 2 次
- 健康检查、日志、报表均可正常读取
